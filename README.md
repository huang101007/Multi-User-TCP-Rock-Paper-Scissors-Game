# Multi-User TCP Rock-Paper-Scissors Game

## 專案簡介
本專案透過 Python 實作基於 TCP 協定的網路連線猜拳遊戲。專案包含基礎的單人對戰版本（玩家與伺服器隨機出拳對戰），以及進階的多人連線版本（支援多位玩家與伺服器主機端同步對戰）。透過此專案，完整演練了底層網路通訊的建立、資料封包的收發，以及多人連線環境下的狀態同步問題。

## 核心功能與系統架構

* **底層通訊建置**
  * 使用 Python 內建 `socket` 模組，以 `AF_INET` 指定 IPv4 位址格式，並以 `SOCK_STREAM` 建立穩定可靠的 TCP 連線通道。

* **多執行緒與連線管理**
  * 在多人連線版本 (`server_multiuser.py`) 中，導入 `threading`模組建立多執行緒架構。
  * 利用獨立的 `handle_client` 執行緒處理不同玩家的初始連線與身分識別（學號/名稱輸入），並使用Dictionary結構統一管理所有連線物件與即時分數。

* **遊戲邏輯與廣播同步機制**
  * 伺服器端負責主導遊戲流程，於每回合向所有客戶端發送 ROUND_START廣播指令，通知玩家出拳。
  * 收集所有出拳資料後，透過Set運算與預設的勝負對照表判定該局勝負（包含平手判定），最終將統合計分板同步回傳給所有玩家，確保各端畫面一致。

* **例外處理與防呆機制**
  * **客戶端防呆：** 針對玩家輸入進行字串預處理（去除空白、轉小寫），若輸入非有效指令（rock/paper/scissors），會直接在客戶端攔截並要求重新輸入，避免傳送無效封包加重伺服器運算負擔。
  * **連線穩定性：** 系統全面導入 `try...except...finally` 錯誤捕捉架構。若發生網路不穩或玩家異常斷線，程式能輸出錯誤資訊而不致崩潰，並確保執行 `close()` 正確釋放 Socket 資源與 Port，讓伺服器能安全回到初始狀態等待下一次連線

## 專案檔案說明
* `server.py`: 基礎單人版伺服器程式，接收單一玩家連線並以隨機方式出拳對戰。
* `client.py`: 玩家端程式，負責發起連線、發送出拳指令並接收即時賽況。
* `server_multiuser.py`: 進階多人版伺服器程式，支援多位玩家連線，並由主機端共同參與對戰與廣播賽況。

## 執行方式
1. 確保所有測試裝置位於同一區域網路下，並確認伺服器裝置的 IP 位址。
2. 修改程式碼中的 `HOST` (或 `SERVER_IP`) 變數為伺服器的實際 IP 位址。
3. **啟動伺服器：**
   * 單人版：在終端機執行 `python server.py`
   * 多人版：在終端機執行 `python server_multiuser.py`
4. **啟動玩家端：**
   * 在終端機執行 `python client.py`，依照提示輸入學號後即可開始遊戲。
